# docs/

Decisiones de arquitectura del proyecto en formato ADR (Architecture Decision Record).

Cada vez que se toma una decisión técnica importante — cambio de stack, patrón de diseño, estrategia de DB, etc. — se documenta aquí.

## ¿Por qué ADRs?

- Las decisiones importantes quedan escritas con su contexto y trade-offs
- El equipo (y la IA) entiende el *por qué* detrás del código
- Si alguien propone revertir una decisión, el ADR muestra qué se consideró antes

## Cómo crear un ADR

1. Copia `ADR_template.md`
2. Nómbralo `ADR-001-nombre-corto.md`
3. Rellena contexto, opciones y decisión
4. Actualiza `Engram.md` con un resumen de la decisión
